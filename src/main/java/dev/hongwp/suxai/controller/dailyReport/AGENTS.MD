# AGENTS.md

# 적용 범위

이 문서는 `dailyReport` 패키지 및 하위 패키지에만 적용한다.

프로젝트 루트 `AGENTS.md`를 기본으로 하며, 본 문서는 Daily Report 기능에 대한 규칙을 정의한다.

---

# 기능 목적

Daily Report 기능은 AI를 이용하여 일일 보고서를 자동 생성하는 기능이다.

특정 시간에 배치를 실행하여 데이터를 수집하고 AI가 분석한 결과를 사람이 읽기 쉬운 게시글 형태의 Markdown 보고서로 생성한다.

현재는 테스트 기능으로 운영하며 일부 대상만 분석한다.

---

# 기능 범위

이 패키지는 다음 기능만 담당한다.

- 일일 배치 실행
- 데이터 수집
- AI 분석
- Markdown 보고서 생성
- 보고서 공유(확장 가능)

다른 기능은 담당하지 않는다.

---

# 패키지 구조

```text
dailyReport
│
├── controller
│   └── DailyReportController
│
├── service
│   ├── DailyReportService
│   ├── AIClient
│   └── ReportWriter
│
├── model
│
└── AGENTS.md
```

---

# 역할

## DailyReportController

- 수동 실행 API를 제공한다.
- 비즈니스 로직을 작성하지 않는다.
- DailyReportService만 호출한다.

## DailyReportService

- 일일보고 전체 프로세스를 제어한다.
- 데이터 수집
- AI 분석 요청
- 보고서 생성
- ReportWriter 호출

## AIClient

- AI 프롬프트 생성
- AI 호출
- AI 응답 처리

## ReportWriter

- Markdown 보고서 생성
- 파일 저장
- 향후 GitHub Issue 등 다른 저장 방식으로 확장 가능하도록 구현한다.

---

# 배치 규칙

- 하루에 한 번 실행한다.
- 실행 시간은 설정 파일에서 관리한다.
- 실행 시간은 코드에 하드코딩하지 않는다.
- 서버가 실행 중일 때만 동작한다.
- 수동 실행 API를 제공할 수 있다.
- 동일 날짜의 보고서는 중복 생성하지 않는다.

---

# 데이터 수집 규칙

- 필요한 데이터는 외부 API를 이용한다.
- 테스트 단계에서는 지정된 일부 대상만 분석한다.
- 분석 대상은 설정 파일에서 관리한다.
- 분석 대상을 코드에 하드코딩하지 않는다.

---

# AI 분석 규칙

AI는 다음 내용을 포함하여 보고서를 생성한다.

- 전체 요약
- 주요 이슈
- 상세 분석
- AI 의견
- 개선 또는 권장 사항

AI 호출 로직은 AIClient에서만 수행한다.

---

# 보고서 규칙

보고서는 Markdown 형식으로 생성한다.

보고서는 사람이 읽기 쉬운 게시글 형태여야 한다.

보고서에는 최소한 다음 내용을 포함한다.

- 보고 일시
- 분석 대상
- 전체 요약
- 주요 이슈
- 상세 분석
- AI 의견

---

# 저장 규칙

DB를 사용하지 않는다.

보고서는 Markdown 파일로 생성한다.

저장 경로는 다음과 같다.

```text
reports/dailyReport/yyyy/MM/yyyy-MM-dd.md
```

동일 날짜 보고서는 하나만 존재해야 한다.

저장 방식은 향후 GitHub Issue, 메일, Slack 등으로 쉽게 확장 가능하도록 구현한다.

---

# 변경 범위

- 다른 패키지의 비즈니스 로직을 수정하지 않는다.
- 새로운 기능은 dailyReport 패키지 내부에서 해결하는 것을 우선한다.
- Common 수정은 반드시 필요한 경우에만 수행한다.
- 다른 패키지 수정이 필요하면 먼저 이유와 영향 범위를 설명한다.

---

# 테스트 규칙

다음 테스트를 작성한다.

## DailyReportService

- 전체 처리 흐름이 정상 수행되는지 확인한다.
- 데이터 수집 → AI 분석 → Markdown 생성 순서가 정상적으로 수행되는지 확인한다.
- 동일 날짜 보고서가 이미 존재하면 중복 생성하지 않는지 확인한다.
- 예외 발생 시 로그를 남기고 정상 종료되는지 확인한다.

## AIClient

- AI 요청 프롬프트가 정상적으로 생성되는지 확인한다.
- AI 응답을 정상적으로 파싱하는지 확인한다.
- AI 응답 실패 시 적절한 예외 처리가 되는지 확인한다.

## ReportWriter

- Markdown 파일이 지정된 경로에 생성되는지 확인한다.
- 파일명이 yyyy-MM-dd.md 형식인지 확인한다.
- 보고서에 필수 항목이 포함되는지 확인한다.
- 동일 날짜 보고서를 중복 생성하지 않는지 확인한다.

## Controller

- 수동 실행 API가 정상 동작하는지 확인한다.
- DailyReportService를 정상 호출하는지 확인한다.

## 통합 테스트

- 실제 외부 API를 이용한 데이터 수집
- 실제 AI를 이용한 보고서 생성
- Markdown 파일 생성

- 데이터 수집 성공

다음 상황에서는 임의로 구현하지 않는다.

- 다른 패키지 수정
- Common 수정
- DB 추가

먼저 이유와 영향 범위를 설명한 후 확인을 요청한다.
