## 1. 프로젝트 개요

### 1.1 배경
한국수자원공사(K-water)가 공개하는 전국 정수장의 수질·유량 데이터는 공공 API로 제공되지만, 일반 시민이 직관적으로 확인할 수 있는 서비스가 부재하다. 수자이는 이 데이터를 실시간으로 수집·시각화하고 AI가 이상징후를 자동 분석함으로써 수질 정보 접근성을 높이는 것을 목적으로 한다.

### 1.2 목표
- 전국 41개 정수장의 수질·유량 데이터를 지정 기간으로 조회
- 기준치 초과 항목을 직관적 색상 경보로 표시
- AI(Groq Llama 3.3 70B)가 이상징후를 종합 분석하여 한국어 리포트 제공
- 비전문가도 이해할 수 있는 수질 정보 대시보드 구성

### 1.3 서비스 대상
- **일반 시민**: 거주 지역 수질 현황 확인
- **환경·수질 관심 사용자**: 전국 정수장 비교 모니터링

## 3. 시스템 범위

### 포함 범위 (In Scope)
- 전국 정수장 목록 조회 및 선택
- 지정 기간 수질 데이터 조회 (pH, 탁도, 잔류염소)
- 지정 기간 유량 데이터 조회 (순간·적산)
- 조회 기간 퀵 버튼 (오늘 / 7일 / 30일 / 90일) 및 날짜 직접 입력
- AI 기반 이상징후 분석 리포트
- 수계 관련 뉴스 티커

### 제외 범위 (Out of Scope)
- 사용자 회원가입·로그인 (추후 검토)
- 데이터 이력 저장 및 시계열 차트 (DB 구축 후 Phase 3)
- 알림·경보 발송 기능 (추후 검토)
- 모바일 앱 (웹 반응형으로 대응 예정)

## 4. 기능 요구사항

### FR-001. 정수장 선택

| 항목 | 내용 |
|------|------|
| 요구사항 ID | FR-001 |
| 요구사항명 | 정수장 선택 |
| 설명 | 사용자가 전국 41개 정수장 중 하나를 선택하면 해당 정수장의 수질·유량 데이터를 조회한다 |
| 우선순위 | 필수 |
| 구현 상태 | 완료 |

- 정수장 목록은 K-water 정수장 목록 API에서 실시간 조회
- 목록은 24시간 캐시하여 API 호출 최소화
- 최초 진입 시 첫 번째 정수장이 자동 선택됨
- 선택 변경 시 수질·유량 탭 데이터 즉시 갱신

### FR-002. 수질 현황 조회

| 항목 | 내용 |
|------|------|
| 요구사항 ID | FR-002 |
| 요구사항명 | 수질 현황 조회 |
| 설명 | 선택된 정수장의 지정 기간 수질 데이터를 카드 형태로 표시한다 (기본 조회 기간: 최근 7일) |
| 우선순위 | 필수 |
| 구현 상태 | 완료 |

**표시 지표 및 정상 기준**

| 지표 | 단위 | 정상 범위 | 주의 | 위험 |
|------|------|-----------|------|------|
| pH | — | 6.5 ~ 8.5 | 6.0~6.5 또는 8.5~9.0 | < 6.0 또는 > 9.0 |
| 탁도 | NTU | ≤ 0.5 | 0.5 ~ 1.0 | > 1.0 |
| 잔류염소 | mg/L | 0.1 ~ 2.0 | 0.05~0.1 또는 2.0~4.0 | < 0.05 또는 > 4.0 |

- 날짜 미선택 시 백엔드에서 현재 시각 기준 최근 1시간 자동 적용
- 데이터는 조회 기간(startDate|endDate)별로 5분 캐시 후 재조회
- 이상 항목이 하나라도 있으면 카드 좌측에 색상 보더 표시

### FR-003. 유량 현황 조회

| 항목 | 내용 |
|------|------|
| 요구사항 ID | FR-003 |
| 요구사항명 | 유량 현황 조회 |
| 설명 | 선택된 정수장의 지정 기간 유량 데이터를 카드 형태로 표시한다 |
| 우선순위 | 필수 |
| 구현 상태 | 완료 |

- 순간유량(㎥/분 등)과 적산유량 구분 표시
- 측정 항목명, 측정값, 단위, 측정시간 표시
- 데이터는 조회 기간(startDate|endDate)별로 5분 캐시 후 재조회

### FR-004. AI 이상징후 분석

| 항목 | 내용 |
|------|------|
| 요구사항 ID | FR-004 |
| 요구사항명 | AI 이상징후 분석 |
| 설명 | 수질·유량 데이터를 Groq AI에 전달하여 이상징후 분석 리포트를 생성한다 |
| 우선순위 | 필수 |
| 구현 상태 | 완료 |

- 분석 항목: 전체 현황 / 이상징후 / 권고사항
- 수질·유량 데이터를 fcltyMngNo 기준으로 조인하여 상관관계 분석
- 결과는 (sujCode|startDate|endDate) 복합 키로 5분 캐시
- AI 모델: Groq Llama 3.3 70B (한국어 전용 출력)

### FR-005. 뉴스 티커

| 항목 | 내용 |
|------|------|
| 요구사항 ID | FR-005 |
| 요구사항명 | 수계 뉴스 티커 |
| 설명 | 수계 관련 주요 이슈를 화면 상단에 스크롤 형태로 표시한다 |
| 우선순위 | 보통 |
| 구현 상태 | 완료 (더미 데이터) |

- 현재: 하드코딩된 7개 뉴스 항목
- Phase 2: 외부 뉴스 API 또는 관리자 등록 방식으로 전환 예정

### FR-006. 조회 기간 선택

| 항목 | 내용 |
|------|------|
| 요구사항 ID | FR-006 |
| 요구사항명 | 조회 기간 선택 |
| 설명 | 날짜 범위를 직접 입력하거나 퀵 버튼으로 선택하여 수질·유량·AI분석 데이터를 해당 기간 기준으로 조회한다 |
| 우선순위 | 보통 |
| 구현 상태 | 완료 |

- 날짜 선택 영역은 정수장 선택바와 탭 사이에 위치
- 퀵 버튼: 오늘 / 7일(기본) / 30일 / 90일
- 날짜 직접 입력: 시작일(`dateFrom`) ~ 종료일(`dateTo`) 날짜 피커
- 조회 버튼 클릭 시 현재 선택된 기간으로 API 재요청
- 날짜 미입력 시 백엔드 기본값(최근 1시간) 적용

## 5. 비기능 요구사항

### NFR-001. 성능

| 항목 | 목표 |
|------|------|
| 정수장 목록 조회 | 첫 요청 3초 이내, 캐시 시 즉시 |
| 수질·유량 조회 | 5초 이내 |
| AI 분석 | 30초 이내 (Groq API 응답 포함) |
| API 타임아웃 | 연결 5초 / 읽기 25초 |

### NFR-002. 가용성
- 외부 API 장애 시 빈 목록 반환 후 사용자에게 안내 (서버 다운 방지)
- 캐시 데이터 존재 시 API 실패와 무관하게 기존 데이터 제공

### NFR-003. 보안
- API 키(K-water, Groq)는 `application.properties`에 관리, `.gitignore` 처리
- 소스코드 저장소에 키 미포함

### NFR-004. 유지보수성
- 정수장 선택, 수질, 유량, AI 분석을 독립 서비스로 분리
- 외부 API 변경 시 클라이언트 클래스만 수정하면 되는 구조
