### UC-001. 대시보드 진입

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 서비스 접속 후 첫 번째 정수장의 수질·유량 현황을 확인한다 |
| 사전 조건 | 없음 |
| 사후 조건 | 첫 번째 정수장의 수질·유량 데이터가 화면에 표시된다 |

**기본 흐름**
1. 사용자가 서비스 URL에 접속한다
2. 시스템이 K-water 정수장 목록 API를 호출한다 (24시간 캐시)
3. 드롭다운에 전국 41개 정수장이 표시된다
4. 조회 기간이 기본값(최근 7일)으로 자동 설정된다
5. 시스템이 첫 번째 정수장(sujCode)으로 수질·유량 API를 자동 호출한다
6. 수질 탭에 데이터가 카드 형태로 표시된다

**예외 흐름**
- 2-1. 정수장 목록 API 호출 실패 → "목록 로드 실패" 메시지 표시
- 5-1. 수질·유량 API 호출 실패 → "데이터가 존재하지 않습니다" 표시

### UC-002. 정수장 변경

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 다른 정수장을 선택하여 해당 정수장의 데이터를 확인한다 |
| 사전 조건 | UC-001 완료 (정수장 목록이 드롭다운에 로드된 상태) |
| 사후 조건 | 선택된 정수장의 수질·유량 데이터가 갱신된다 |

**기본 흐름**
1. 사용자가 드롭다운에서 다른 정수장을 선택한다
2. 시스템이 선택된 정수장의 sujCode 및 현재 조회 기간으로 수질 API를 호출한다
3. 시스템이 선택된 정수장의 sujCode 및 현재 조회 기간으로 유량 API를 호출한다
4. 수질·유량 탭의 카드 데이터가 갱신된다

**예외 흐름**
- 2-1. 캐시 유효 시(동일 sujCode+기간) → API 미호출, 캐시 데이터 즉시 반환
- 3-1. 데이터 없는 정수장 선택 → "데이터가 존재하지 않습니다" 표시

### UC-003. 수질 현황 확인

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 선택된 정수장의 지정 기간 pH·탁도·잔류염소 수치와 이상 여부를 확인한다 |
| 사전 조건 | 정수장이 선택된 상태 |
| 사후 조건 | 없음 |

**기본 흐름**
1. 사용자가 수질 탭을 클릭한다 (기본 탭)
2. 수질 데이터 카드가 표시된다
3. 각 카드에 시설명, 측정시간, pH·탁도·잔류염소 수치가 표시된다
4. 기준치 초과 항목은 주황(주의) 또는 빨강(위험) 색으로 강조된다

**색상 판정 규칙**

| 상태 | 카드 좌측 보더 | 수치 색상 |
|------|------------|---------|
| 정상 | 없음 | 흰색 |
| 주의 | 주황 | 주황 |
| 위험 | 빨강 | 빨강 |

### UC-004. 유량 현황 확인

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 선택된 정수장의 지정 기간 유량 측정값을 확인한다 |
| 사전 조건 | 정수장이 선택된 상태 |
| 사후 조건 | 없음 |

**기본 흐름**
1. 사용자가 유량 탭을 클릭한다
2. 유량 데이터 카드가 표시된다
3. 각 카드에 시설명, 측정 항목명, 측정값, 단위, 측정시간이 표시된다
4. 순간유량은 초록 배지, 적산유량은 주황 배지로 구분된다

### UC-005. AI 이상징후 분석 실행

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 현재 수질·유량 데이터를 AI가 종합 분석한 리포트를 확인한다 |
| 사전 조건 | 정수장이 선택된 상태 |
| 사후 조건 | AI 분석 리포트가 화면에 표시된다 |

**기본 흐름**
1. 사용자가 AI 분석 탭을 클릭한다
2. 사용자가 "분석 실행" 버튼을 클릭한다
3. 스켈레톤 로딩 화면이 표시된다
4. 시스템이 현재 조회 기간의 수질·유량 데이터를 조회하여 Groq AI에 분석 요청한다
5. AI가 전체 현황 / 이상징후 / 권고사항을 한국어로 반환한다
6. 리포트가 화면에 표시된다

**예외 흐름**
- 4-1. 동일 (sujCode+기간) 5분 캐시 유효 → AI 재호출 없이 캐시 결과 즉시 반환
- 4-2. Groq API 키 미설정 → "API 키를 설정해 주세요" 안내 메시지 표시
- 4-3. Groq API 오류 → 오류 메시지 표시

### UC-007. 데이터 자동 캐시 갱신 (시스템)

| 항목 | 내용 |
|------|------|
| 액터 | 시스템 |
| 목적 | 캐시 만료 시 다음 요청에서 최신 데이터를 자동으로 갱신한다 |
| 사전 조건 | 캐시 TTL이 만료된 상태 |
| 사후 조건 | 새 데이터로 캐시가 갱신된다 |

**캐시 TTL 정책**

| 데이터 | 캐시 키 | TTL |
|--------|---------|-----|
| 정수장 목록 | 단일 (전체 목록) | 24시간 |
| 수질 데이터 | sujCode\|startDate\|endDate | 5분 |
| 유량 데이터 | sujCode\|startDate\|endDate | 5분 |
| AI 분석 결과 | sujCode\|startDate\|endDate | 5분 |

### UC-008. 조회 기간 변경

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 날짜 범위를 변경하여 해당 기간의 수질·유량·AI분석 데이터를 조회한다 |
| 사전 조건 | UC-001 완료 (대시보드 초기 로드 완료) |
| 사후 조건 | 변경된 기간의 데이터가 조회된다 |

**기본 흐름 A — 퀵 버튼 사용**
1. 사용자가 오늘 / 7일 / 30일 / 90일 퀵 버튼 중 하나를 클릭한다
2. 시작일(dateFrom)·종료일(dateTo)이 자동 계산된다
3. 클릭한 퀵 버튼이 활성(active) 상태로 변경된다

**기본 흐름 B — 날짜 직접 입력**
1. 사용자가 시작일(dateFrom) 또는 종료일(dateTo) 날짜 피커를 클릭하여 날짜를 선택한다
2. 날짜 입력 시 퀵 버튼 active 상태가 해제된다

**공통 흐름 (A·B 이후)**
1. 사용자가 "조회" 버튼을 클릭한다
2. 시스템이 현재 선택된 정수장(sujCode)과 startDate·endDate를 포함하여 API를 호출한다
3. 캐시 히트 시 즉시 반환, 미스 시 외부 API 재호출 후 5분 캐시 갱신

**예외 흐름**
- 날짜 미입력 시 → 백엔드 기본값(현재 시각 기준 최근 1시간) 자동 적용
