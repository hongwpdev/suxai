### UC-001. 대시보드 진입

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 서비스 접속 후 첫 번째 정수장의 수질·유량 현황을 확인한다 |
| 사전 조건 | 없음 |
| 사후 조건 | 첫 번째 정수장의 수질·유량 데이터가 화면에 표시된다 |

**기본 흐름**
1. 사용자가 서비스 URL에 접속한다
2. 시스템이 K-water 정수장 목록 API를 호출한다 (24시간 캐시)
3. 드롭다운에 전국 41개 정수장이 표시된다
4. 시스템이 첫 번째 정수장(sujCode)으로 수질·유량 API를 자동 호출한다
5. 수질 탭에 데이터가 카드 형태로 표시된다

**예외 흐름**
- 2-1. 정수장 목록 API 호출 실패 → "목록 로드 실패" 메시지 표시
- 4-1. 수질·유량 API 호출 실패 → "데이터가 존재하지 않습니다" 표시

### UC-002. 정수장 변경

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 다른 정수장을 선택하여 해당 정수장의 데이터를 확인한다 |
| 사전 조건 | UC-001 완료 (정수장 목록이 드롭다운에 로드된 상태) |
| 사후 조건 | 선택된 정수장의 수질·유량 데이터가 갱신된다 |

**기본 흐름**
1. 사용자가 드롭다운에서 다른 정수장을 선택한다
2. 시스템이 선택된 정수장의 sujCode로 수질 API를 호출한다
3. 시스템이 선택된 정수장의 sujCode로 유량 API를 호출한다
4. 수질·유량 탭의 카드 데이터가 갱신된다

**예외 흐름**
- 2-1. 캐시 유효 시 → API 미호출, 캐시 데이터 즉시 반환
- 3-1. 데이터 없는 정수장 선택 → "데이터가 존재하지 않습니다" 표시

### UC-003. 수질 현황 확인

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 선택된 정수장의 pH·탁도·잔류염소 수치와 이상 여부를 확인한다 |
| 사전 조건 | 정수장이 선택된 상태 |
| 사후 조건 | 없음 |

**기본 흐름**
1. 사용자가 수질 탭을 클릭한다 (기본 탭)
2. 수질 데이터 카드가 표시된다
3. 각 카드에 시설명, 측정시간, pH·탁도·잔류염소 수치가 표시된다
4. 기준치 초과 항목은 주황(주의) 또는 빨강(위험) 색으로 강조된다

**색상 판정 규칙**

| 상태 | 카드 좌측 보더 | 수치 색상 |
|------|------------|---------|
| 정상 | 없음 | 흰색 |
| 주의 | 주황 | 주황 |
| 위험 | 빨강 | 빨강 |

### UC-004. 유량 현황 확인

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 선택된 정수장의 유량 측정값을 확인한다 |
| 사전 조건 | 정수장이 선택된 상태 |
| 사후 조건 | 없음 |

**기본 흐름**
1. 사용자가 유량 탭을 클릭한다
2. 유량 데이터 카드가 표시된다
3. 각 카드에 시설명, 측정 항목명, 측정값, 단위, 측정시간이 표시된다
4. 순간유량은 초록 배지, 적산유량은 주황 배지로 구분된다

### UC-005. AI 이상징후 분석 실행

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 현재 수질·유량 데이터를 AI가 종합 분석한 리포트를 확인한다 |
| 사전 조건 | 정수장이 선택된 상태 |
| 사후 조건 | AI 분석 리포트가 화면에 표시된다 |

**기본 흐름**
1. 사용자가 AI 분석 탭을 클릭한다
2. 사용자가 "분석 실행" 버튼을 클릭한다
3. 스켈레톤 로딩 화면이 표시된다
4. 시스템이 수질·유량 데이터를 조회하여 Groq AI에 분석 요청한다
5. AI가 전체 현황 / 이상징후 / 권고사항을 한국어로 반환한다
6. 리포트가 화면에 표시된다

**예외 흐름**
- 4-1. 5분 캐시 유효 → AI 재호출 없이 캐시 결과 즉시 반환
- 4-2. Groq API 키 미설정 → "API 키를 설정해 주세요" 안내 메시지 표시
- 4-3. Groq API 오류 → 오류 메시지 표시

### UC-006. 데이터 수동 새로고침

| 항목 | 내용 |
|------|------|
| 액터 | 일반 사용자 |
| 목적 | 현재 탭의 데이터를 수동으로 재조회한다 |
| 사전 조건 | 데이터가 한 번 이상 로드된 상태 |
| 사후 조건 | 최신 데이터로 갱신된다 (캐시 유효 시 캐시 데이터 반환) |

**기본 흐름**
1. 사용자가 "조회" 버튼을 클릭한다
2. 상단 프로그레스바가 표시된다
3. 스켈레톤 로딩 후 최신 데이터가 표시된다

### UC-007. 데이터 자동 캐시 갱신 (시스템)

| 항목 | 내용 |
|------|------|
| 액터 | 시스템 |
| 목적 | 캐시 만료 시 다음 요청에서 최신 데이터를 자동으로 갱신한다 |
| 사전 조건 | 캐시 TTL이 만료된 상태 |
| 사후 조건 | 새 데이터로 캐시가 갱신된다 |

**캐시 TTL 정책**

| 데이터 | TTL |
|--------|-----|
| 정수장 목록 | 24시간 |
| 수질 데이터 | 5분 |
| 유량 데이터 | 5분 |
| AI 분석 결과 | 5분 (sujCode별) |
