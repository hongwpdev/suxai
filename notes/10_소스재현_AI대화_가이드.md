## 1. 이 문서의 사용 목적

정의서 5종(요구사항·유스케이스·아키텍처·화면·API)은 시스템의 **무엇(What)**을 설명하지만,
실제 코드로 옮기는 **어떻게(How)**는 추가 질문 없이는 구현자(또는 AI)가 임의로 결정하게 된다.

이 문서는 **정의서 → AI 코드 생성** 과정에서 어떤 질문을 해야 원본과 동일한 소스가 재현되는지를
파일별로 정리한 가이드다.

#### ② `QualityApiClient.java` / `FlowApiClient.java` — K-water 응답 JSON 파싱

정의서 05 § 2.1에 중첩 구조와 단건/복수 분기 처리가 추가되었으나,
실제 파싱 로직은 AI가 직접 구현해야 한다.

**필요한 추가 질문**
```
K-water API 응답 JSON은 아래와 같이 중첩되어 있어:

{
  "response": {
    "header": { "resultCode": "00" },
    "body": {
      "items": {
        "item": [ ... ]   // 단건이면 Map, 복수이면 List<Map>
      }
    }
  }
}

파싱 조건:
1. header.resultCode가 "00"이 아니면 빈 리스트 반환
2. items가 null이거나 빈 문자열("")이면 빈 리스트 반환
3. item이 Map이면 List.of(map)으로 감싸서 처리
4. item이 List<Map>이면 그대로 사용
5. occrrncDt는 YYYYMMDDhh(10자리) → "YYYY-MM-DD HH시" 형식으로 변환
6. 필드 값이 null이면 빈 문자열("")로 처리

위 조건을 만족하는 extractItems() 메서드와 toRecord() 변환 메서드를 포함하여 작성해줘.
```

#### ③ `GroqApiClient.java` — system 프롬프트 전문

정의서 05 § 2.4에 system 프롬프트 전문이 수록되어 있다. 그대로 사용하면 된다.

**AI 요청 시 명시할 내용**
```
system 프롬프트는 정확히 아래 문자열을 사용해줘 (한 줄 연결):
"당신은 수처리 전문가입니다. 반드시 한국어(한글)로만 답변하세요. 한자(殺菌, 濃度 등 한문)를 절대 사용하지 마세요. 한자 대신 반드시 한글을 사용하세요: 살균, 농도, 능력, 부족 등. 영어 단어도 가능하면 한국어로 바꾸세요."

isConfigured() 메서드도 포함시켜줘:
- apiKey가 blank이거나 "YOUR_GROQ_API_KEY_HERE" 이면 false 반환
- 그 외는 true 반환
```

#### ④ `AnalysisService.java` — buildPrompt() 전문

정의서 05 § 2.4에 user 프롬프트 전체 구조가 수록되어 있다.

**AI 요청 시 명시할 내용**
```
buildPrompt(List<QualityRecord> wqList, List<FlowRecord> flList) 메서드는 아래 규칙으로 작성해줘:

1. wqList를 fcltyMngNo(QualityRecord.getId()) 기준으로 Map<String, QualityRecord>로 인덱싱
   (중복 시 첫 번째 유지: (a, b) -> a)
2. flList를 fcltyMngNo 기준으로 Map<String, List<FlowRecord>>로 groupingBy
3. 교집합(joinedIds), wq만(wqOnlyIds), fl만(flOnlyIds) 세 그룹으로 분류
4. 각 그룹은 최대 10개만 포함 (.stream().limit(10))
5. 각 그룹이 비어있으면 해당 섹션 생략
6. wqList와 flList 모두 비어있으면 "현재 수집된 데이터가 없습니다." 삽입

프롬프트 말미 분석 지시 문구는 아래와 같이 정확히 사용해줘:
(05_API_명세서.md § 2.4 buildPrompt 섹션 참조)
```

#### ⑥ `index.html` — 수질 판정 임계값 및 뉴스 티커 JS

정의서 04 § SCR-001-A에 판정 기준표가 추가되었다. 프론트엔드 구현 시 아래를 명시한다.

**필요한 추가 질문**
```
수질 카드의 색상 판정 로직은 아래 임계값을 JavaScript로 구현해줘:

const phCls  = (ph < 5.8 || ph > 9.0)   ? 'c-crit' : (ph < 6.5 || ph > 8.5)   ? 'c-warn' : 'c-ok';
const tbCls  = tb > 1.0                  ? 'c-crit' : tb > 0.5                  ? 'c-warn' : 'c-ok';
const clCls  = (cl < 0.05 || cl > 4.0)  ? 'c-crit' : (cl < 0.1 || cl > 2.0)   ? 'c-warn' : 'c-ok';

카드 클래스: 세 항목 중 하나라도 c-crit → 카드 crit / c-warn 존재 → 카드 warn / 모두 c-ok → 카드 ok

뉴스 티커는 GET /api/news 응답의 "text" 필드를 읽어서 표시해줘.
(응답 필드명: level, text — content 아님)
```

## 3. 전체 AI 대화 흐름 (권장 순서)

정의서를 가지고 처음부터 소스를 재구현한다면 아래 순서로 진행한다.

```
Step 1. 프로젝트 구조 생성
  → "Spring Boot 3.3 / Java 17 / Maven 프로젝트를 생성해줘.
     패키지: dev.hongwp.suxai
     레이어: controller / service / client / model / config"

Step 2. 모델 클래스 생성 (4개)
  → 05_API명세서 § 1.1~1.5 응답 필드를 붙여넣고 각 모델 요청
  → NewsItem은 "text" 필드임을 명시

Step 3. AppConfig (RestTemplate 타임아웃)
  → "connectTimeout 5000ms, readTimeout 25000ms"

Step 4. K-water 클라이언트 3개
  → URL·파라미터·중첩 JSON 파싱 구조 전달 (05 § 2.1~2.3)
  → 단건/복수 분기, null 처리, occrrncDt 변환 명시

Step 5. Service 3개 (Quality/Flow/Facility)
  → 03 § 5 캐시 전략 (캐시 키·TTL) 전달

Step 6. GroqApiClient
  → system 프롬프트 전문 직접 붙여넣기 (05 § 2.4)
  → isConfigured() 조건 명시

Step 7. AnalysisService (buildPrompt 포함)
  → 조인 로직·limit(10)·분기 조건 명시
  → 프롬프트 말미 분석 지시 문구 직접 붙여넣기

Step 8. ApiController
  → 5개 엔드포인트 및 뉴스 7개 항목 텍스트 붙여넣기

Step 9. index.html
  → 04 § 화면 설계서 레이아웃·색상·판정 임계값 전달
  → Vanilla JS, fetch API, 스켈레톤 카드·뉴스 티커 구현 요청
```

## 5. 검증 체크리스트

재구현 후 아래 항목을 확인하여 원본과의 일치 여부를 검증한다.

- [ ] `GET /api/news` 응답에서 JSON 키가 `text`인가 (`content` 아님)
- [ ] `GET /api/waterQuality?sujCode=318`으로 조회 시 44초 미만 응답
- [ ] K-water 단건 응답 시 정상 파싱되는가 (List.of() 래핑 확인)
- [ ] AI 분석 결과에 한자(殺菌, 濃度 등)가 포함되지 않는가
- [ ] pH 6.2인 카드가 `warn` 클래스(주황 테두리)로 표시되는가
- [ ] pH 5.5인 카드가 `crit` 클래스(빨강 테두리)로 표시되는가
- [ ] 잔류염소 0.03 mg/L인 카드가 `crit`으로 표시되는가
- [ ] 캐시 TTL 5분 이내 동일 요청이 API 재호출 없이 응답되는가
- [ ] Spring Boot 시작 로그에 빈(Bean) 등록 오류가 없는가
