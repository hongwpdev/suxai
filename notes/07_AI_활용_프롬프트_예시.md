## 개요

수자이 프로젝트는 아이디어 구상부터 산출물 문서 작성, 프로토타입 개발, Spring Boot 전환까지 전 과정에서 AI를 활용하였다. 아래는 단계별 실제 요청 방식과 결과물 예시다.

## STEP 1. 아이디어 구체화 및 주제 선정

### 요청 예시

```
우리 팀은 한국수자원공사 유지관리 사업에 근무하는 팀원들로 구성된 학습조직이야.
AI를 활용한 미니 프로젝트 주제를 선정하려고 하는데,
K-water 업무와 연관성이 높고 공공 API를 활용할 수 있는 주제를 추천해줘.
```

### AI 답변 (요약)

```
추천 주제: 실시간 수질·유량 이상징후 감지 대시보드

- K-water가 공개하는 공공데이터 API(수질/유량/정수장 목록)를 활용할 수 있음
- AI(LLM)를 연동하면 수치 데이터를 자동으로 분석하고 한국어 리포트로 제공 가능
- 업무 연계성이 높고 실제 데이터를 다루기 때문에 학습 효과가 큼
```

## STEP 4. 시스템 아키텍처 정의서 작성 (03_시스템_아키텍처_정의서.md)

### 요청 예시

```
아래 기술 스택으로 시스템 아키텍처 정의서를 작성해줘.

- 프론트엔드: Vanilla JS + HTML (SPA)
- 백엔드: Spring Boot 3.3 / Java 17
- AI: Groq API (LLaMA 3.3 70B)
- 외부 데이터: K-water 공공 API (수질, 유량, 정수장 목록)

레이어는 Controller → Service → Client 구조로 하고,
Service와 Client는 1:1 대응 네이밍(XxxService ↔ XxxApiClient)으로 설계해줘.
캐시 전략도 서비스별로 TTL을 다르게 적용해줘.
```

### 결과물 핵심 내용

시스템 구성도(ASCII), 레이어 구조, 데이터 흐름(정수장 목록/수질/AI 분석), 캐시 전략 표, 설계 결정 사항 정리.
주요 설계 결정으로 DB 미사용 이유, sujCode 필수화 이유, Groq 선택 이유, Vanilla JS 선택 이유 명시.

## STEP 7. Python 프로토타입 개발 (app.py, data.py)

### 요청 예시

```
수자이 대시보드를 Streamlit으로 만들어줘.

- 수질 측정소 10개, 유량 측정소 10개 샘플 데이터 포함
```

`data.py`에 `wq_score()`, `fl_score()`, `get_status()`, `jitter()` 함수 구현.
`app.py`에 Streamlit 기반 다크 테마 대시보드, 뉴스 티커, 수질/유량 탭, 2.5초 자동 갱신 구현.

## STEP 8. Spring Boot 전환 — 백엔드 개발

### 요청 예시

```
Python 프로토타입을 Spring Boot 3.3 / Java 17로 전환해줘.

구조:
- Controller → Service → Client 레이어 분리
- XxxService ↔ XxxApiClient 1:1 대응 네이밍
- 각 서비스는 ConcurrentHashMap으로 캐시 관리 (TTL: 수질/유량/AI 5분, 정수장 목록 24시간)

클라이언트:
- QualityApiClient: K-water 수질 API 호출, occrrncDt 날짜 포맷 변환
- FlowApiClient: K-water 유량 API 호출, divType(M→순간, D→적산) 변환
- FacilityApiClient: K-water 정수장 목록 API 호출
- GroqApiClient: Groq AI API 호출 (OpenAI 호환, 시스템 프롬프트 한글 전용)

RestTemplate 타임아웃은 연결 5초, 읽기 25초로 설정해줘.
API 키는 application.properties에서 관리하고 .gitignore 처리해줘.
```

### 결과물 핵심 내용

`AppConfig.java`에 RestTemplate 빈 등록 및 타임아웃 설정.
`ApiController.java`에 5개 REST 엔드포인트 구현.
각 Client 클래스에서 공공 API JSON 파싱 및 모델 변환 처리.
`AnalysisService.java`에서 수질·유량 데이터를 `fcltyMngNo` 기준으로 조인 후 프롬프트 생성.

## STEP 9. AI 분석 프롬프트 개선

### 요청 예시

```
아래 AI 분석 프롬프트를 개선해줘.

현재 문제:
1. AI 응답에 한자(殺菌, 濃度)가 섞여 나옴
2. 데이터 없는 경우 AI가 임의로 문구를 만들어냄
3. 출력 형식이 매번 달라서 프론트에서 파싱하기 어려움

개선 요청:
- 시스템 프롬프트에 한글 전용 출력 + 한자 금지 명시
- 데이터 없는 경우의 답변 문구("이상징후 없음", "특이사항 없음")를 프롬프트에 고정
- 출력 형식을 **1. 전체 현황** / **2. 이상징후** / **3. 권고사항** 으로 고정
- 수질 기준값(pH 6.5~8.5, 탁도 0.5 NTU 이하, 잔류염소 0.1~2.0 mg/L)을 프롬프트에 포함
```

### 결과물 핵심 내용

`GroqApiClient.java` 시스템 프롬프트에 한자 금지 + 한글 전용 지시 추가.
`AnalysisService.java` 프롬프트 말미에 출력 형식 및 기준값 명시.
결과 없음 케이스 답변 문구를 프롬프트에 고정하여 일관성 확보.

## STEP 10. 프론트엔드 개발 (index.html)

### 요청 예시

```
Spring Boot에서 제공하는 REST API를 호출하는 단일 페이지 대시보드를 Vanilla JS로 만들어줘.

- 다크 테마 (배경 #111318, 카드 #181b22)
- 상단: 로딩 프로그레스바
- 헤더: 서비스명 + 부제목
- 뉴스 티커: /api/news 호출, 40초 무한 스크롤, 호버 시 일시 정지
- 정수장 드롭다운: /api/facilities 호출, 선택 시 수질·유량 즉시 재조회
- 탭: 수질(/api/waterQuality) / 유량(/api/waterFlow) / AI 분석(/api/analysis)
- 로딩 중: 상단 프로그레스바 + 스켈레톤 카드 shimmer 애니메이션
- 수질 카드: pH·탁도·잔류염소 색상 경보 (정상/주의/위험)
- AI 분석: **볼드** 마크다운을 <b> 태그로 변환해서 렌더링
```

### 결과물 핵심 내용

`static/index.html` 단일 파일로 전체 SPA 구현.
`loadFacilities()`, `fetchWaterQuality()`, `fetchWaterFlux()`, `fetchAnalysis()` 함수로 API 호출 분리.
스켈레톤 카드 shimmer 애니메이션, 상단 프로그레스바, 수질 경보 색상 판정 로직 포함.
AI 분석 리포트는 `replace(/\*\*(.*?)\*\*/g, '<b>$1</b>')` 처리로 마크다운 볼드 렌더링.
