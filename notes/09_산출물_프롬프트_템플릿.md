# 산출물 프롬프트 템플릿

| 항목 | 내용 |
|------|------|
| 작성일 | 2026-05-24 |
| 수정일 | 2026-05-24 (시뮬레이션 테스트 후 입력 항목 보완) |
| 목적 | 어떤 프로젝트에서든 동일한 품질의 산출물을 만들 수 있도록 프롬프트를 템플릿화한 문서 |
| 사용 방법 | `[ ]` 안의 내용만 프로젝트에 맞게 바꿔서 AI에게 그대로 붙여넣기 |

---

## 사용 순서

산출물은 아래 순서대로 만든다.
앞 단계 산출물을 다음 단계 프롬프트의 입력으로 활용하면 대화 없이 바로 생성 가능하다.

```
TEMPLATE-01. 요구사항 정의서
      ↓ (결과물을 입력으로 사용)
TEMPLATE-02. 유스케이스 정의서
      ↓
TEMPLATE-03. 시스템 아키텍처 정의서
      ↓
TEMPLATE-04. 화면 설계서
      ↓
TEMPLATE-05. API 명세서
```

## TEMPLATE-01. 요구사항 정의서

### 입력 정보 (채워야 할 항목)

| 항목 | 설명 | 예시 |
|------|------|------|
| [프로젝트명] | 서비스 이름 | 수자이(水Xai) |
| [프로젝트 배경] | 왜 만드는지 | K-water 공공 API는 있지만 시민이 보기 쉬운 서비스가 없다 |
| [핵심 목표] | 무엇을 하는 서비스인지 1~2줄 | 수질·유량 데이터를 실시간 시각화하고 AI가 이상징후를 분석 |
| [서비스 대상] | 누가 쓰는지 | 일반 시민, 환경 관심 사용자 |
| [이해관계자 목록] | 관련된 주체와 관심사 | 최종 사용자/데이터 제공자/AI 서비스/개발·운영 |
| [핵심 기능 목록] | 반드시 있어야 할 기능들 (우선순위 포함) | 정수장 선택(필수), 수질 현황(필수), 유량 현황(필수), AI 분석(필수), 뉴스 티커(보통) |
| [구현 상태] | 각 기능의 현재 완료 여부 | 완료 / 진행중 / 미착수 |
| [데이터 판정 기준] | 정상·주의·위험 판단 기준값 | pH 6.5~8.5 정상, 탁도 0.5 NTU 이하 정상, 잔류염소 0.1~2.0 mg/L 정상 |
| [제외 범위] | 이번에 안 하는 것 | 회원가입, DB 이력 저장, 모바일 앱 |
| [NFR 성능 수치] | 응답 시간 목표 및 타임아웃 | 수질·유량 5초 이내, AI 분석 30초 이내, 연결 타임아웃 5초, 읽기 타임아웃 25초 |
| [기술 제약] | 사용 가능한 기술·환경 제약 | DB 없음, 외부 API 의존, AI 무료 티어 토큰 제한 |
| [Phase 2 기능 목록] | 다음 단계에서 할 기능 | 모바일 반응형, 지역 필터, 뉴스 실데이터 연동 |
| [Phase 3 기능 목록] | DB 구축 후 할 기능 | 시계열 차트, 이상징후 이력, AI 리포트 누적 저장 |

### 프롬프트

## 활용 팁

**처음 프로젝트라면**
TEMPLATE-01부터 순서대로 진행한다.
01 결과물 → 02 입력, 01+03 결과물 → 05 입력처럼 앞 단계 산출물을 다음 단계에 그대로 붙여넣으면 된다.

**이미 진행 중인 프로젝트라면**
현재 단계의 템플릿부터 시작하고, 이전 단계 산출물이 있으면 입력으로 활용한다.

**팀원과 함께 쓸 때**
`[ ]` 항목을 팀이 함께 채운 후 프롬프트를 AI에게 전달하면, 누가 요청하더라도 동일한 수준의 산출물이 나온다.

**입력 항목을 더 채울수록 완성도가 높아진다**
시뮬레이션 테스트 결과, 입력 항목이 많을수록 기존 산출물과의 일치율이 높아졌다.
특히 데이터 판정 기준값, 캐시 TTL, 색상 코드, 설계 결정 이유는 꼭 채워넣을 것을 권장한다.
