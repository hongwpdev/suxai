# PM 요구사항 → 산출물 자동 생성 프롬프트 체인 가이드

| 항목 | 내용 |
|------|------|
| 작성일 | 2026-06-03 |
| 목적 | PM이 자연어로 요구사항을 작성하면, 각 단계의 AI 프롬프트가 연쇄 실행되어 산출물 5종이 순서대로 자동 생성되는 체인 구조 정의 |
| 대상 | Claude, ChatGPT 등 대화형 AI |
| 기존 문서와의 차이 | 09_템플릿: 사람이 `[ ]` 항목을 채워야 함 → 이 문서: PM 자연어만 있으면 AI가 알아서 추출·작성 |

---

## 1. 전체 체인 구조

```
┌─────────────────────────────────────────────────┐
│  PM 요구사항 (자연어, 형식 자유)                   │  ← PM이 작성하는 유일한 입력
└────────────────────┬────────────────────────────┘
                     │ STEP 1 프롬프트
                     ▼
         01_요구사항_정의서.md
                     │ STEP 2 프롬프트  (01 전체 포함)
                     ▼
         02_유스케이스_정의서.md
                     │ STEP 3 프롬프트  (01 + 02 전체 포함)
                     ▼
         03_시스템_아키텍처_정의서.md
                     │ STEP 4 프롬프트  (01 + 03 전체 포함)
                     ▼
         04_화면_설계서.md
                     │ STEP 5 프롬프트  (03 + 04 전체 포함)
                     ▼
         05_API_명세서.md
```

**핵심 원칙**: 각 단계의 출력 전체를 다음 단계 프롬프트에 그대로 붙여넣는다.
추가 설명 없이 "산출물 전문 붙여넣기 → 프롬프트 실행"으로 진행한다.

---

## 2. STEP 0 — PM 요구사항 작성

PM은 아래 항목을 자연어로 채운다. **완벽하지 않아도 된다.** 불확실한 항목은 "미정" 또는 생략.

```
## 프로젝트 개요
- 프로젝트명:
- 한 줄 설명 (서비스가 무엇을 하는가):
- 왜 만드는가 (배경·문제 상황):
- 주요 사용자:

## 핵심 기능 (꼭 있어야 하는 것)
1.
2.
3.

## 추가 기능 (나중에 해도 되는 것)
1.
2.

## 하지 않을 것 (명시적 제외)
-

## 기술 환경
- 사용 언어·프레임워크:
- 외부 API·서비스 연동:
- 배포 환경:
- DB 사용 여부:

## 성능·제약 조건
- 응답 시간 목표:
- 보안·인증 요구사항:
- 기타 제약:
```

> PM이 위 항목을 모두 채울 필요는 없다. AI가 빈 항목을 합리적으로 추론하거나 "확인 필요" 표시로 채운다.

## 3. STEP 1 — 요구사항 정의서 생성

**입력**: STEP 0의 PM 요구사항

**프롬프트** (아래 전문을 복사하고, `===` 사이에 PM 요구사항을 붙여넣어 실행)

```
당신은 소프트웨어 요구사항 분석가입니다.
아래 PM 요구사항을 읽고, 마크다운 형식의 요구사항 정의서를 작성해주세요.

===PM 요구사항 시작===
[STEP 0에서 작성한 PM 요구사항 전체 붙여넣기]
===PM 요구사항 끝===

작성 규칙:
1. 문서 상단에 메타정보 표 포함 (프로젝트명, 버전, 작성일)
2. 기능 요구사항은 FR-001부터 번호를 부여하고 아래 표 형식으로 작성
   | 번호 | 구분 | 요구사항명 | 설명 | 우선순위 | 비고 |
   구분: MVP / Phase2 / Phase3
   우선순위: 높음 / 보통 / 낮음
3. 비기능 요구사항(NFR)은 별도 절에 작성 (성능/보안/가용성/유지보수성)
4. 기술 제약사항은 표로 정리
5. PM 요구사항에 없지만 시스템 구현에 필요한 요구사항은 AI가 추론하여 추가하되,
   비고 열에 "[AI 추론]" 표시
6. 불확실한 항목은 비고에 "[확인 필요]" 표시

출력 형식: 마크다운 전문 (코드블록 없이 바로 마크다운 출력)
```

**STEP 1 완료 후**: 출력된 내용을 `docs/01_요구사항_정의서.md`로 저장

## 4. STEP 2 — 유스케이스 정의서 생성

**입력**: STEP 1 출력물 (01_요구사항_정의서.md 전체)

**프롬프트**

```
당신은 소프트웨어 분석가입니다.
아래 요구사항 정의서를 읽고, 마크다운 형식의 유스케이스 정의서를 작성해주세요.

===요구사항 정의서 시작===
[01_요구사항_정의서.md 전체 내용 붙여넣기]
===요구사항 정의서 끝===

작성 규칙:
1. 문서 상단에 메타정보 표 포함
2. 액터 정의: 시스템과 상호작용하는 주체를 모두 식별 (사람·외부 시스템 포함)
3. 유스케이스 목록 표
   | UC 번호 | 유스케이스명 | 주 액터 | 관련 FR | 우선순위 |
   UC-001부터 번호 부여
4. 유스케이스 상세 (FR 우선순위 "높음" 항목 우선)
   - 사전 조건
   - 기본 흐름 (번호 매긴 단계)
   - 예외 흐름 (오류 케이스)
   - 사후 조건
5. FR과 UC의 교차 참조 표 포함

출력 형식: 마크다운 전문
```

**STEP 2 완료 후**: `docs/02_유스케이스_정의서.md`로 저장

## 5. STEP 3 — 시스템 아키텍처 정의서 생성

**프롬프트**

```

아래 요구사항 정의서와 유스케이스 정의서를 읽고, 시스템 아키텍처 정의서를 작성해주세요.

===요구사항 정의서 시작===

===요구사항 정의서 끝===

===유스케이스 정의서 시작===
[02_유스케이스_정의서.md 전체 내용 붙여넣기]
===유스케이스 정의서 끝===

작성 규칙:
1. 시스템 구성도: ASCII 텍스트 다이어그램으로 클라이언트·서버·외부시스템 관계 표현
2. 기술 스택 표: 계층(프론트엔드/백엔드/DB/외부서비스)별로 기술·버전·용도 정리
3. 레이어 구조: 패키지/디렉토리 트리 형태로 표현
4. 데이터 흐름: 주요 유스케이스별 시퀀스를 텍스트로 표현 (화살표 사용)
5. 캐시 전략: 캐시 대상, 캐시 키, TTL, 저장소를 표로 정리
6. 외부 API 연동: URL, 인증 방식, 응답 형식 명세
7. 주요 설계 결정 사항: 결정과 이유를 표로 정리
8. 확장 고려사항 (Phase 2/3)

출력 형식: 마크다운 전문
```

## 8. 체인 실행 요령

### 8.1 대화 방식

각 STEP은 **새 대화(New Chat)**에서 실행하거나, 같은 대화에서 연속 실행해도 된다.
같은 대화에서 실행하면 AI가 맥락을 유지하므로 더 일관된 출력이 나온다.

```
[권장 방식]
대화 1개에서 STEP 1 ~ STEP 5 순서대로 실행
→ AI가 앞 단계 내용을 참고하여 더 정확하게 생성
```

### 8.2 출력 품질이 낮을 때

AI가 항목을 빠뜨리거나 형식이 맞지 않을 때 아래 보완 프롬프트를 추가로 실행:

```
위 [문서명]에서 아래 항목이 누락되었습니다. 해당 부분만 보완해주세요.
- [누락된 항목 1]
- [누락된 항목 2]
```

### 8.3 실제 사용 예시 (suxai 프로젝트)

```
PM 요구사항 입력:
  "K-water 공공 API를 이용해서 전국 정수장의 수질·유량 데이터를 보여주는
   웹 대시보드를 만들어줘. 실시간으로 이상징후를 감지하고 AI로 분석해줬으면 해.
   회원가입 같은 건 필요 없고, Spring Boot + 외부 AI API 사용해서
   DB 없이 간단하게 만들어줘."

STEP 1 실행 → 01_요구사항_정의서.md 생성 (FR-001~005 MVP, FR-006~013 Phase2/3)
STEP 2 실행 → 02_유스케이스_정의서.md 생성 (UC-001~007)
STEP 3 실행 → 03_시스템_아키텍처_정의서.md 생성 (Spring Boot 레이어, 캐시 전략)
STEP 4 실행 → 04_화면_설계서.md 생성 (수질·유량·AI 탭, 색상 정의)
STEP 5 실행 → 05_API_명세서.md 생성 (5개 엔드포인트, K-water/Groq 연동)
```

| 구분 | 09_산출물_프롬프트_템플릿 | 11_PM요구사항_문서자동체인 (이 문서) |
|------|------------------------|--------------------------------------|
| 입력 방식 | 사람이 `[ ]` 항목을 직접 채움 | PM 자연어 → AI가 추출·작성 |
