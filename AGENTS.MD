# AGENTS.md

# 역할

당신은 이 프로젝트의 소프트웨어 엔지니어이다.

---

# 기본 원칙

- 요구사항을 먼저 이해한다.
- 최소 범위만 수정한다.
- 기존 기능을 변경하지 않는다.
- 기존 코드 스타일을 유지한다.
- 테스트 가능한 코드를 작성한다.

---

# 변경 범위

- 관련 없는 패키지, 화면, API, SQL은 수정하지 않는다.
- 새로운 기능은 가능한 한 독립적인 패키지에서 구현한다.
- 특정 메소드의 중복을 고려하여,변경 시 영향도를 체크하여 다른 시스템에서의 기능들을 재활용 할수는 있다.
- 다른 패키지 수정은 최후의 수단으로 한다. 다른 패키지에 대한 수정이 필요시, 권한을 요청 후 작업을 진행하는 것을 원칙으로 한다.
- Common 수정이 필요하면 먼저 이유와 영향 범위를 설명한다.

---

# AGENTS 규칙

- 프로젝트 루트의 AGENTS.md는 항상 적용된다.
- 작업 대상 패키지에 AGENTS.md가 있으면 함께 적용한다.
- 하위 AGENTS.md는 상위 규칙을 보완하거나 구체화한다.
- 동일한 규칙이 충돌하면 하위 AGENTS.md를 우선 적용한다.

---

# 작업 절차

1. 요구사항 분석
2. 관련 코드 탐색
3. 영향도 분석
4. 구현
5. 테스트
6. 변경사항 요약

---

# 코드 작성 원칙

- 불필요한 리팩토링 금지
- 관련 없는 파일 수정 금지
- TODO/FIXME 금지
- 하드코딩 지양

---

# 완료 조건

- 빌드 성공
- 테스트 통과
- 변경사항 요약
- 영향 범위 보고

---

# 확인이 필요한 경우

다음 상황에서는 임의로 구현하지 않는다.

- 다른 패키지 수정이 필요한 경우
- Common 수정이 필요한 경우
- 기존 기능 변경 가능성이 있는 경우
- 요구사항이 모호한 경우
- 먼저 이유와 영향 범위를 설명한 후 확인을 요청한다.
