------------------------------
파일별 설명
------------------------------

[pom.xml]
- Maven 빌드 설정 파일
- Spring Boot 3.3.0 / Java 17
- 의존성:
  · spring-boot-starter-web  : REST API + 내장 Tomcat 서버
  · spring-boot-starter-test : 테스트 라이브러리


[SuxaiApplication.java]
- Spring Boot 진입점 (main 메서드)
- @SpringBootApplication으로 컴포넌트 자동 스캔·설정


------------------------------
config/
------------------------------

[config/AppConfig.java]
- RestTemplate Bean 등록 및 타임아웃 설정
  · 연결 타임아웃: 5,000 ms
  · 읽기 타임아웃: 25,000 ms
- 설정 이유: K-water API는 sujCode 없이 호출 시 응답 시간 44초 이상 소요


------------------------------
controller/
------------------------------

[controller/ApiController.java]
- REST API 엔드포인트 정의 (기본 경로: /api)
- 제공 API:
  · GET /api/facilities    : 전국 정수장 목록 (41개)
  · GET /api/waterQuality  : 수질 현황 (sujCode 파라미터, 선택)
  · GET /api/waterFlow     : 유량 현황 (sujCode 파라미터, 선택)
  · GET /api/analysis      : AI 이상징후 분석 (sujCode 파라미터, 선택)
  · GET /api/news          : 수계 뉴스 티커 목록
- sujCode 미입력 시 application.properties의 kwater.suj.code 기본값 사용
- 뉴스 데이터는 컨트롤러 내 하드코딩 (Phase 2에서 실데이터 전환 예정)


------------------------------
client/  (외부 API 호출 전담 계층)
------------------------------

네이밍 규칙: XxxApiClient ↔ XxxService (1:1 대응)

[client/QualityApiClient.java]
- K-water 수질 API 호출 및 JSON 파싱
- 엔드포인트: https://apis.data.go.kr/B500001/rwis/waterQuality/list
- 조회 범위: 현재 시각 기준 최근 1시간 (stDt~edDt)
- 응답 파싱: Map<String, Object> → WaterQualityRecord 변환
- occrrncDt 포맷 변환: YYYYMMDDhh → YYYY-MM-DD hh시

[client/FlowApiClient.java]
- K-water 유량 API 호출 및 JSON 파싱
- 엔드포인트: https://apis.data.go.kr/B500001/rwis/waterFlux/waterFlux
- 응답 파싱: Map<String, Object> → FlowRecord 변환
- divType 변환: M → 순간 / D → 적산

[client/FacilityApiClient.java]
- K-water 정수장 목록 API 호출 및 JSON 파싱
- 엔드포인트: https://apis.data.go.kr/B500001/rwis/waterQuality/fcltylist/codelist
- 요청 파라미터: fcltyDivCode=2, numOfRows=100, pageNo=1
- 응답: 전국 41개 정수장 (fcltyMngNm, sujCode)

[client/GroqApiClient.java]
- Groq AI API 호출 (OpenAI 호환 형식)
- 엔드포인트: https://api.groq.com/openai/v1/chat/completions
- 모델: llama-3.3-70b-versatile
- max_tokens: 2048
- 시스템 프롬프트: 한국어 전용 출력, 한자 사용 금지


------------------------------
model/  (데이터 모델)
------------------------------

[model/WaterQualityRecord.java]
- K-water 수질 API 응답 매핑 모델
- 필드:
  · id          : 시설관리번호 (fcltyMngNo)
  · facilityName: 시설명 (fcltyMngNm)
  · address     : 시설 주소 (fcltyAddr)
  · divName     : 구분명 - 정수/원수 (liIndDivName)
  · measuredAt  : 측정시간 포맷팅 (occrrncDt)
  · phVal/phUnit: pH 값·단위
  · tbVal/tbUnit: 탁도 값·단위 (NTU)
  · clVal/clUnit: 잔류염소 값·단위 (mg/L)

[model/FlowRecord.java]
- K-water 유량 API 응답 매핑 모델
- 필드:
  · id          : 시설관리번호 (fcltyMngNo)
  · facilityName: 시설명 (fcltyNm)
  · description : 측정 항목 설명 (dataItemDesc)
  · value       : 측정값 (dataVal)
  · unit        : 단위 (itemUnit)
  · measuredAt  : 측정시간
  · divType     : 유형 - 순간 / 적산

[model/FacilityInfo.java]
- 정수장 목록 API 응답 매핑 모델
- 필드:
  · sujCode     : 수계 코드
  · facilityName: 정수장 명칭
  · address     : 주소 (현재 미제공, 빈 문자열)

[model/AnalysisResult.java]
- AI 분석 결과 모델
- 필드:
  · analysis   : AI 분석 리포트 본문 (마크다운 **볼드** 포함)
  · analyzedAt : 분석 기준 시각 (YYYY-MM-DD HH:mm)
  · wqCount    : 분석에 사용된 수질 데이터 건수
  · flCount    : 분석에 사용된 유량 데이터 건수

[model/NewsItem.java]
- 뉴스 티커 항목 모델
- 필드:
  · level  : 심각도 (crit / warn / ok)
  · content: 뉴스 본문


------------------------------
service/  (비즈니스 로직·캐시 관리)
------------------------------

네이밍 규칙: XxxService ↔ XxxApiClient (1:1 대응)

[service/QualityService.java]
- 수질 데이터 캐시 관리 (sujCode별 독립 캐시)
- 캐시 저장소: ConcurrentHashMap<String(sujCode), List<WaterQualityRecord>>
- TTL: 5분
- 캐시 만료 시 QualityApiClient 호출하여 갱신

[service/FlowService.java]
- 유량 데이터 캐시 관리 (sujCode별 독립 캐시)
- 캐시 저장소: ConcurrentHashMap<String(sujCode), List<FlowRecord>>
- TTL: 5분
- 캐시 만료 시 FlowApiClient 호출하여 갱신

[service/FacilityService.java]
- 정수장 목록 캐시 관리 (전체 목록 단일 캐시)
- TTL: 24시간
- 캐시 만료 시 FacilityApiClient 호출하여 갱신

[service/AnalysisService.java]
- AI 이상징후 분석 요청 및 결과 캐시 (sujCode별 독립 캐시)
- 캐시 저장소: ConcurrentHashMap<String(sujCode), AnalysisResult>
- TTL: 5분
- 분석 절차:
  1. QualityService·FlowService에서 수질·유량 데이터 조회
  2. fcltyMngNo 기준으로 두 데이터셋 조인
  3. 프롬프트 구성 (통합 데이터 / 수질만 / 유량만 3섹션)
  4. GroqApiClient로 AI 분석 요청
  5. 결과 캐시 저장

  · kwater.suj.code = 318  (기본 정수장 코드 — 파주정수장)
  · groq.api.key = (Groq API 키)

[application.properties.example]  ← GitHub 공개용 템플릿
- 실제 키 없이 형식만 기재
- 복사 후 application.properties로 사용

[static/index.html]
- 단일 페이지 대시보드 (SPA)
- 구성:
  · 헤더: 서비스명, 부제목
  · 뉴스 티커: 무한 좌→우 스크롤 (40초 주기, 호버 시 정지)
  · 정수장 선택바: 드롭다운 (41개), 선택 시 수질·유량 즉시 갱신
  · 탭: 수질 / 유량 / AI 분석
  · 로딩 UX: 상단 프로그레스바 + 스켈레톤 카드 shimmer
- 주요 함수:
  · loadFacilities()   : 정수장 목록 로드 및 드롭다운 구성
  · onFacilityChange() : 정수장 변경 시 데이터 재조회
  · fetchWaterQuality(): /api/waterQuality 호출 → 수질 카드 렌더링
  · fetchWaterFlux()   : /api/waterFlow 호출 → 유량 카드 렌더링
  · fetchAnalysis()    : /api/analysis 호출 → AI 리포트 렌더링
- 수질 경보 기준:
  · pH        : 6.5~8.5 정상 / 6.0~6.5·8.5~9.0 주의 / 그 외 위험
  · 탁도       : ≤0.5 NTU 정상 / 0.5~1.0 주의 / >1.0 위험
  · 잔류염소   : 0.1~2.0 mg/L 정상 / 0.05~0.1·2.0~4.0 주의 / 그 외 위험
